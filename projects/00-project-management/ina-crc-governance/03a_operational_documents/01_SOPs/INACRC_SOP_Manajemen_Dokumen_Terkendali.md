## 1. TUJUAN

Menetapkan sistem manajemen dokumen terkendali untuk memastikan integritas, keabsahan, ketersediaan, dan keamanan semua dokumen kualitas di lingkungan Indonesia Clinical Research Center (INA-CRC) serta jejaring Clinical Research Unit (CRU) seluruh Indonesia.

## 2. RUANG LINGKUP

SOP ini berlaku untuk semua dokumen terkendali di sistem manajemen mutu INA-CRC, termasuk:
- Dokumen kebijakan, prosedur, dan instruksi kerja
- Dokumen regulator dan perizinan
- Dokumen uji klinis dan studi
- Dokumen pelatihan dan kompetensi
- Dokumen supplier dan vendor
- Template dan form terstandar
- Dokumen eksternal yang dikendalikan

## 3. DEFINISI

| Istilah | Definisi |
|---------|----------|
| **Dokumen Terkendali** | Dokumen yang statusnya, versinya, dan distribusinya dikelola secara formal untuk mencegah penggunaan dokumen yang tidak sah |
| **Master Document** | Dokumen asli yang disetujui dan disimpan sebagai single source of truth |
| **Dokumen Kadaluwarsa** | Dokumen yang tidak lagi berlaku dan telah diganti dengan versi terbaru |
| **Dokumen Eksternal** | Dokumen dari luar organisasi yang mempengaruhi sistem manajemen mutu |
| **Dokumen Reference** | Dokumen yang digunakan sebagai referensi tetapi tidak menjadi bagian dari sistem formal |
| **QMS** | Quality Management System - Sistem Manajemen Mutu |
| **Document Control System** | Sistem elektronik untuk mengelola dokumen terkendali |

## 7. PROSEDUR

### 7.1 Pembuatan Dokumen Baru

**7.1.1 Proses Workflow:**
```
PIC Dokumen → Draft → Review Unit → Review Mutu → Approval → Input System → Distribusi
    ↓              ↓          ↓           ↓         ↓           ↓            ↓
  1 hari        3-5 hari    2-3 hari    1-2 hari   1 hari      1 hari       Langsung
```

**7.1.2 Contoh Implementasi - Pembuatan SOP Baru:**
```
Skenario: Membuat SOP untuk "Handling Serious Adverse Event Reporting"

Step 1: Initiation (PIC: Pharmacovigilance Officer, Timeline: 1 hari)
1.1. Identifikasi kebutuhan berdasarkan gap analysis
1.2. Submit Document Creation Request (Form QMS-001)
1.3. Get approval dari Kepala Unit

Step 2: Drafting (PIC: Pharmacovigilance Officer, Timeline: 3 hari)
2.1. Gunakan template SOP standar INA-CRC
2.2. Include regulatory references (BPOM, ICH-GCP)
2.3. Define timeline yang jelas (24-hour reporting)
2.4. Sertakan flowchart proses reporting

Step 3: Internal Review (PIC: Kepala Unit, Timeline: 2 hari)
3.1. Review substansi dan implementability
3.2. Cek alignment dengan proses existing
3.3. Validasi dengan PIC terkait (IT, Legal)

Step 4: Quality Review (PIC: Manajer Mutu, Timeline: 1 hari)
4.1. Review format dan struktur
4.2. Cek compliance dengan template standar
4.3. Validasi referensi dan cross-references

Step 5: Approval (PIC: Kepala Unit, Timeline: 1 hari)
5.1. Final review dan sign-off
5.2. Assign document number: INACRC-SOP-PV-001
5.3. Set effective date dan review date

Step 6: System Input (PIC: Document Controller, Timeline: 1 hari)
6.1. Input ke QMS system
6.2. Upload PDF dan source file
6.3. Set distribution list
6.4. Generate automatic notifications

Step 7: Training (PIC: Training Coordinator, Timeline: 1 minggu)
7.1. Schedule training session
7.2. Distribusi ke user yang relevan
7.3. Document training completion
```

### 7.3 Review dan Update Dokumen

**7.3.1 Schedule Review:**
- **Annual Review:** Semua SOP dan manual
- **Biennial Review:** Policy dan strategic documents
- **Trigger Review:** Perubahan regulasi, audit finding, incident

**7.3.2 Proses Review:**
```
System Alert → PIC Review → Assessment → Action Required → Implementation
     ↓              ↓           ↓            ↓                 ↓
  30 hari         7 hari      3 hari      7-14 hari         Langsung
```

**7.3.3 Contoh Review Implementation:**
```
Skenario: Annual Review SOP INACRC-SOP-001 (Registrasi Proposal)

Step 1: System Alert (System, 30 hari sebelum due date)
1.1. Automatic email ke PIC SOP
1.2. Alert ke Manajer Mutu dan Kepala Unit
1.3. Update status di dashboard menjadi "Due for Review"

Step 2: Content Review (PIC: Admin Team, 5 hari)
2.1. Check apakah proses masih relevan
2.2. Validasi compliance dengan regulasi terbaru
2.3. Collect feedback dari user (survey 1 minggu)
2.4. Identifikasi gaps atau improvement opportunities

Step 3: Assessment (PIC: Manajer Mutu, 2 hari)
3.1. Evaluasi apakah perlu update minor atau major
3.2. Minor update (V1.0 → V1.1): Format, typo, clarification
3.3. Major update (V1.1 → V2.0): Proses change, regulatory update

Step 4: Update Implementation (Jika diperlukan)
4.1. Minor update: Revise dan distribute tanpa training ulang
4.2. Major update: Full review cycle seperti dokumen baru

Step 5: Documentation (Document Controller, 1 hari)
5.1. Update version number dan effective date
5.2. Archive old version dengan catatan perubahan
5.3. Update Master Document Register
```

### 7.4 Distribusi dan Akses Dokumen

**7.4.1 Metode Distribusi:**
- **Electronic Distribution:** Primary method melalui QMS system
- **Hard Copy:** Hanya untuk documents requiring original signatures
- **Controlled Areas:** Printed versions di specific locations

**7.4.2 Access Control Levels:**
| Level | Access Type | Document Types | Users |
|-------|-------------|----------------|-------|
| **Public** | View Only | Annual Reports, Public Info | All stakeholders |
| **Internal** | View/Download | SOPs, Work Instructions | All INA-CRC staff |
| **Restricted** | View/Download/Print | Sensitive data, Contracts | Authorized users only |
| **Confidential** | View Only (Secure) | Personal data, Confidential | Very limited access |

**7.4.3 Contoh Implementasi Akses Control:**
```
Skenario: Managing Access to Clinical Trial Documents

Document Types dan Access Levels:
1. Public Documents:
   - INA-CRC Annual Report → All stakeholders
   - GCP Training Schedule → Public website
   - General Guidelines → Accessible via portal

2. Internal Documents:
   - All SOPs (INACRC-SOP-XXX) → All INA-CRC staff
   - Work Instructions → Staff by department
   - Training Materials → Trainees only

3. Restricted Documents:
   - Clinical Trial Agreements → Legal, Management only
   - Budget Information → Finance, Management
   - Audit Reports → Management, Audit Team

4. Confidential Documents:
   - Patient Data in CRFs → Study team only
   - Personal Information → HR, Individual only
   - Investigator Contracts → Legal, PI, Sponsor only
```

### 7.5 Dokumen Eksternal Terkendali

**7.5.1 Jenis Dokumen Eksternal:**
- **Regulatory Documents:** BPOM regulations, KMK, International guidelines
- **Customer Documents:** Sponsor protocols, agreements
- **Supplier Documents:** Certificates, specifications
- **Partner Documents:** CRU agreements, MOUs

**7.5.2 Proses Control Dokumen Eksternal:**
```
Reception → Evaluation → Registration → Control → Monitoring
    ↓           ↓           ↓          ↓           ↓
  1 hari     3-5 hari    1 hari     Ongoing     Quarterly
```

**7.5.3 Contoh Control Dokumen Eksternal:**
```
Skenario: Managing New BPOM Regulation

Step 1: Reception (PIC: Regulatory Affairs, 1 hari)
1.1. Download new BPOM regulation from official website
1.2. Verify authenticity dan publication date
1.3. Initial assessment of impact to INA-CRC

Step 2: Evaluation (PIC: Regulatory Team, 3-5 hari)
2.1. Detailed review of new requirements
2.2. Gap analysis against current processes
2.3. Identify required changes to SOPs/forms
2.4. Create impact assessment report

Step 3: Registration (PIC: Document Controller, 1 hari)
3.1. Assign control number: INACRC-EXT-BPOM-001-2025
3.2. Input to external document register
3.3. Upload to QMS system
3.4. Set review schedule (sesuai lifecycle dokumen)

Step 4: Implementation (PIC: Relevant Units, 2-4 minggu)
4.1. Update affected SOPs
4.2. Revise forms dan templates
4.3. Conduct training jika perlu
4.4. Update system references

Step 5: Monitoring (PIC: Regulatory Affairs, Ongoing)
5.1. Track compliance dengan regulation baru
5.2. Monitor untuk amendments atau updates
5.3. Annual review untuk relevance check
```

### 7.6 Dokumen Kadaluwarsa dan Arsip

**7.6.1 Retention Schedule:**
| Document Type | Retention Period | Archive Location |
|---------------|------------------|------------------|
| **Quality Records** | 15 tahun | Digital Archive |
| **Clinical Trial Documents** | 25 tahun | Secure Digital Archive |
| **Training Records** | 7 tahun | HR Database |
| **Audit Reports** | 10 tahun | QMS Archive |
| **Regulatory Documents** | Permanent | Regulatory Archive |
| **Financial Records** | 10 tahun | Finance Archive |

**7.6.2 Proses Archiving:**
```
Identify Obsolete → Mark as Obsolete → Move to Archive → Update Register → Verify Backup
        ↓                    ↓                  ↓              ↓               ↓
     System Alert          1 hari             1-3 hari       1 hari          1 hari
```

**7.6.3 Contoh Archiving Process:**
```
Skenario: Archiving SOP V1.0 setelah update ke V2.0

Step 1: Obsolete Marking (System, Automatic)
1.1. System automatically marks V1.0 as "Obsolete"
1.2. Send notification to all document users
1.3. Prevent any new distribution of V1.0

Step 2: Archive Process (Document Controller, 1 hari)
2.1. Move V1.0 to archive folder
2.2. Add archive metadata:
    - Archive date
    - Reason for archiving (version update)
    - Access restrictions
2.3. Update Master Document Register

Step 3: Digital Preservation (IT, 1 hari)
3.1. Create backup copy in secure archive server
3.2. Verify file integrity
3.3. Update archive index
3.4. Set retention reminder (25 tahun untuk SOP klinis)

Step 4: Access Control (Document Controller, 1 hari)
4.1. Remove V1.0 from active document library
4.2. Restrict access to V1.0 (read-only untuk audit)
4.3. Update search index to prioritize V2.0
```

### 7.7 Document System Management

**7.7.1 QMS System Features:**
- **Version Control:** Automatic versioning dengan history tracking
- **Access Control:** Role-based permissions
- **Search Function:** Full-text search dengan filters
- **Notifications:** Automatic alerts untuk review/expiry
- **Audit Trail:** Complete tracking semua document activities
- **Reporting:** Comprehensive dashboards dan reports

**7.7.2 Backup dan Recovery:**
- **Daily Backup:** Incremental backup ke secure server
- **Weekly Backup:** Full backup ke offsite location
- **Annual Archive:** Long-term preservation di secure archive
- **Recovery Testing:** Quarterly recovery drill
- **Disaster Recovery:** Hot standby site untuk critical documents

**7.7.3 System Monitoring:**
```
Daily: System health check, backup verification
Weekly: Access log review, performance monitoring
Monthly: Storage optimization, usage analytics
Quarterly: Security audit, vulnerability assessment
Annually: Full system audit, disaster recovery test
```
