## Report Best Practices

### DO's:
- [ ] Use specific numbers and data points
- [ ] Be honest about challenges and setbacks
- [ ] Propose solutions for identified issues
- [ ] Quantify metrics wherever possible
- [ ] Maintain professional, factual tone
- [ ] Include all required sections
- [ ] Proofread for clarity and grammar
- [ ] Submit on time (monthly reports by 5th)
- [ ] Attach referenced documentation
- [ ] Track time accurately

### DON'Ts:
- [ ] Use vague language ("some progress", "good results")
- [ ] Hide challenges or sugarcoat setbacks
- [ ] Exceed time allocation without explanation
- [ ] Skip required sections
- [ ] Use emotional language
- [ ] Overpromise without delivering
- [ ] Submit late without communication
- [ ] Forget attachments
- [ ] Leave out financial calculations
- [ ] Miss support requests that could help
