# Advanced Analysis Plan for Synthea Pilot Data

This document outlines the steps and considerations for performing advanced analyses on the synthetic pilot data generated by Synthea, as required by the study design.

## 1. Sequence Analysis for Behavioral Patterns

**Objective:** To identify common sequences of patient behaviors and events (e.g., medication adherence, blood pressure monitoring, telemedicine encounters) that may correlate with clinical outcomes or engagement levels.

**Relevant Synthea Output Files:**
- `observations.csv`: Contains timestamped records of vital signs (like BP) and other measurements.
- `medications.csv`: Records medication prescriptions and fills, which can be used to infer adherence.
- `encounters.csv` (or `telehealth_encounters.csv` after processing): Records patient interactions, representing telemedicine engagement.
- `conditions.csv`: Used to identify the hypertension cohort and related diagnoses.

**Approach:**
1.  **Event Data Extraction:**
    *   For each patient (from `patients.csv`), extract a chronologically ordered list of events.
    *   Define event types: `BP_MEASUREMENT`, `MED_FILL`, `TELEMED_ENCOUNTER`, `CONDITION_ONSET`, etc.
    *   Ensure each event has `patient_id` and `timestamp`.
2.  **Sequence Construction:**
    *   Group events by `patient_id` and order them by `timestamp`. This will form the "sequence" for each patient.
3.  **Sequence Mining:**
    *   Apply sequence mining algorithms to discover frequent patterns, rules, or motifs within these event sequences.
    *   **Tools:** Python libraries such as `PrefixSpan-py` or `SPMF` (a Java-based tool with Python wrappers available) are suitable.
    *   **Focus:** Look for patterns that precede outcomes (e.g., successful BP control) or distinguish different patient engagement phenotypes.

## 2. Time Series Analysis for Temporal Trends

**Objective:** To analyze the temporal dynamics of key clinical metrics, particularly blood pressure, and identify underlying trends, seasonality, or responses to simulated interventions.

**Relevant Synthea Output Files:**
- `observations.csv`: Crucial for blood pressure readings (LOINC codes: `8480-6` for systolic, `8462-4` for diastolic).
- `patients.csv`: For patient demographics (e.g., age, gender) for cohort stratification.

**Approach:**
1.  **Data Extraction and Filtering:**
    *   From `observations.csv`, filter for relevant blood pressure measurements.
    *   Convert `DATE` column to datetime objects.
2.  **Time Series Construction (Per Patient):**
    *   For each patient, create a time series of their systolic and diastolic BP readings.
    *   Handle irregular measurement intervals: Resample data to a fixed frequency (e.g., daily, weekly mean/median) and interpolate missing values if appropriate for the analysis.
3.  **Trend and Seasonality Detection:**
    *   **Tools:** Python libraries like `pandas`, `statsmodels`, and `scikit-learn` for time series modeling (ARIMA, Exponential Smoothing, Prophet).
    *   **Methods:**
        *   Decomposition of time series into trend, seasonal, and residual components.
        *   Autocorrelation and Partial Autocorrelation Functions (ACF/PACF) to understand temporal dependencies.
        *   Regression models with time-based features to model linear or non-linear trends.
4.  **Event-Based Analysis:**
    *   Correlate BP trends with other events (e.g., medication changes from `medications.csv`, telemedicine encounters) to assess impact.

## 3. Social Network Analysis (If Applicable)

**Objective:** To model hypothetical relationships or interactions within the telemedicine program and analyze network structures, centrality, and influence.

**Applicability:** Synthea does *not* natively generate social network data. This analysis is applicable only if you decide to **simulate** patient-to-patient or patient-to-provider interaction data or infer connections based on shared characteristics (e.g., all patients seen by the same provider).

**Relevant Synthea Output Files (for inference/simulation):**
- `encounters.csv`: Reveals which patients interact with which providers.
- `providers.csv`: Provider demographics.
- `organizations.csv`: Healthcare facility information.

**Approach (Requires Data Simulation/Inference):**
1.  **Define Nodes and Edges:**
    *   **Nodes:** Patients and/or providers.
    *   **Edges (Simulated/Inferred):**
        *   **Patient-Provider:** Based on `encounters.csv` (patient `X` had an encounter with provider `Y`).
        *   **Patient-Patient (Simulated):** Can be simulated based on shared providers, geographic proximity (`patients.csv` zip codes), or random connections for a pilot.
2.  **Network Construction:**
    *   **Tools:** Python library `NetworkX` is excellent for building and analyzing graphs.
    *   Create a graph representation of the network.
3.  **Network Analysis:**
    *   Calculate centrality measures (e.g., degree, betweenness, closeness) to identify influential nodes.
    *   Detect communities or clusters within the network.
    *   Analyze network density, path length, and other structural properties.

## 4. Mixed Methods Integration of Qualitative Field Notes and Quantitative Observational Data

**Objective:** To develop and test your mixed-methods integration strategies by using the quantitative Synthea data as a backdrop for simulated qualitative findings and refining your observational protocol.

**Relevant Synthea Output Files:**
- All quantitative Synthea CSVs (`patients.csv`, `conditions.csv`, `observations.csv`, `medications.csv`, `encounters.csv`).

**Approach:**
1.  **Quantitative Data Archetypes:**
    *   Based on your Synthea data, identify distinct patient groups or "archetypes." For example:
        *   "Consistently adherent with good BP control."
        *   "Intermittently adherent with fluctuating BP."
        *   "Poorly adherent with high BP."
        *   "High technology engagement but poor clinical outcome."
    *   Use basic descriptive statistics or clustering algorithms on relevant Synthea data fields to define these archetypes.
2.  **Simulated Qualitative Field Notes:**
    *   For each quantitative archetype identified, **create a few mock qualitative field notes**. These notes should describe hypothetical observations of patient behaviors, reported barriers (e.g., "patient mentioned device connection issues," "expressed forgetfulness about medication," "cited cost concerns"), or contextual factors that are consistent with that archetype.
    *   **Purpose:** These are *simulated* notes to test your **coding scheme and integration process**, not to generate actual qualitative data from Synthea.
3.  **Process Integration Testing:**
    *   **Test Coding Scheme:** Apply your planned qualitative coding scheme (as detailed in your `RMRE-Module-4-hypertension_telemedicine_study_design.md`) to these mock field notes.
    *   **Refine Protocol:** Identify any ambiguities or gaps in your coding scheme.
    *   **Practice Triangulation:** Practice mapping your coded qualitative themes from the mock notes back to the quantitative metrics and archetypes from Synthea. This helps ensure your integration framework (e.g., convergent, sequential) is feasible and robust.
    *   **Hypothesis Generation:** Use this exercise to generate refined hypotheses about observed behaviors and their underlying reasons, which can then be explored in a real-world study.
